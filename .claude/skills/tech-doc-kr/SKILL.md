# tech-doc-kr - 한국어 기술 문서 교정 (v1.0)

실제 문서 교정에서 독자 피드백으로 검증된 규칙을 적용한다.
규칙 원문: `references/rules.md` (6대 규칙) / `references/calque-blacklist.md` (직역 관용구 SSOT) / `references/special-chars.md` (특수문자 치환표).

## 0. 불변 조건 (철칙 - 모든 모드 공통)

1. **의미/사실/수치/고유명사 불변.** 교정은 표현만 바꾼다.
2. **코드블록/표/링크 보존.** 코드펜스 개수와 내용은 유지가 기본. 레포 실코드 인용은 특수문자까지 소스와 일치시킨다 (special-chars.md 예외 규칙).
3. **문체 register 유지.** 반말 노트는 반말로, 존댓말 문서는 존댓말로.
4. **전수 원칙.** 패턴 1곳 탐지 = 대상 문서 전체(요청 범위가 문서군이면 문서군 전체)에서 같은 패턴 전수 검색 후 일괄 처리.

## 1. 입력 파악

- 대상: 파일 경로(들) 또는 붙여넣은 텍스트. 파일이면 먼저 전체를 읽는다.
- 문서 타입 감지 (교정 가중치가 달라진다):

| 타입 | 판별 힌트 | 가중치 |
|---|---|---|
| PR 본문 | PR/커밋 언급, 변경 요약 구조 | 간결성 + 코드 실명. 형식 검사는 가볍게 |
| README | 설치/사용법/뱃지 | 압축 라벨 제목 검사 + 특수문자 + 링크 검증 비중 크게 |
| docs/기술 노트 | 절 번호, 코드 인용 다수 | 6대 규칙 전부. 코드 인용 보존 규칙 필수 |
| 기술 블로그 | 시리즈/독자 호칭 | 용어/형식/코드 정합 위주. 문체/리듬 윤문은 이 스킬 범위 밖임을 안내 |

- 코드 인용이 있으면 인용 원본 레포 경로를 확인해 둔다 (보존 판별에 필요).

## 2. 모드 선택 (AskUserQuestion, 기본값 = 파이프라인)

시작하며 사용자에게 묻는다. 단, **입력이 1,500자 미만이면 경량을 기본으로 제안**한다.

- **파이프라인 (기본)**: 탐지/교정/검증을 별도 에이전트로 분리. 문서군 교정이나 처음 교정하는 긴 문서에 적합.
- **경량**: 오케스트레이터가 references를 직접 로드해 혼자 수행. 짧은 입력(PR 본문 등), 재교정, 빠른 확인용.

## 3-A. 파이프라인 모드

1. **탐지** - `tech-doc-kr:tell-detector` 에이전트 호출. 입력: 대상 파일 경로(또는 텍스트), 문서 타입. 출력: 패턴별 탐지 리포트 (위치/원문/분류/제안).
2. **리포트 검토** - 오케스트레이터가 리포트를 훑고 오탐(유지 목록 해당, 실코드 인용 등)을 걸러낸 뒤 승인 목록을 만든다. 판단이 갈리는 항목(자생 은유 유지 여부, 절 삭제)은 사용자에게 물어본다.
3. **교정** - `tech-doc-kr:doc-rewriter` 에이전트 호출. 입력: 대상 + 승인된 탐지 목록. 불변 조건 준수하에 전수 교체.
4. **검증** - `tech-doc-kr:doc-verifier` 에이전트 호출. verify.py 실행 + 블랙리스트 잔존 재스캔 + 교정 전후 코드펜스 대조. 실패 항목은 3으로 되돌린다 (최대 2회).
5. **최종 보고** - 아래 출력 형식으로 오케스트레이터가 정리.

## 3-B. 경량 모드

오케스트레이터가 직접, 순서대로:

1. references 3종 로드.
2. 특수문자 전수 조사(verify.py --scan) -> 치환표 적용 (코드 펜스는 언어/실코드 여부 보고 처리).
3. 블랙리스트 항목 전수 검색 -> 문맥 맞는 대안으로 교체 (rules.md 1~3항).
4. 형식/구조 검사 (rules.md 4~5항): 문답 프레임, 압축 라벨 제목, 중복 절, 죽은 참조.
5. verify.py로 기계 검증 -> 잔존 0 확인.

## 4. 검증 스크립트 위치

`verify.py`는 이 스킬 베이스 디렉터리 기준 `../../../scripts/verify.py` (플러그인 루트 `scripts/`).
찾을 수 없으면 같은 검사를 인라인 파이썬으로 수행한다 (비키보드 문자 스캔, 중괄호+한글 피드백 마커, 상대링크 존재, 코드펜스 md5).

## 5. 출력 형식 (최종 보고)

1. **패턴별 변경 건수 표** - 분류(직역 관용구/개념어/실명/특수문자/형식/구조)별 건수.
2. **대표 before -> after** - 분류당 1~2개.
3. **판단 보류 목록** - 자생 은유 등 사용자 결정이 필요한 항목.
4. **verify 결과** - 잔존 0 여부, 링크/코드펜스 상태.
5. 새로 발견된 직역 패턴이 있으면 **calque-blacklist.md에 추가 제안** (SSOT 갱신).

## 6. 하지 않는 것

- 내용 추가/삭제를 동반한 재작성 (요약, 증보) - 별도 작업.
- 문체/리듬 일반 윤문 - 이 스킬의 범위 밖.
- 영어 문서 교정 - 이 스킬은 한국어 기술 문서 전용.
- 유지 목록(calque-blacklist.md 하단) 항목을 어색하다고 임의 교체하는 것.
