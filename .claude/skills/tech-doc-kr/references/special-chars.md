# 특수문자 치환표 - 키보드로 칠 수 있는 문자만 남긴다

비키보드 특수문자가 많다는 것 자체가 "사람이 타이핑하지 않았다"는 신호다.
목표: 문서의 모든 문자가 ASCII + 한글(+ 코드 인용 예외)이 되게 한다.

## 기계 치환표

| 대상 | 치환 | 비고 |
|---|---|---|
| — (em dash), – (en dash) | `-` / 범위는 `~` | "A — B"는 문맥에 따라 쉼표/마침표로 풀어 쓰는 게 더 자연스러울 수 있음 |
| · (가운뎃점) | `/` 또는 `, ` | 짝 결합(A/B 분리)은 `/`, 나열은 쉼표 |
| -> 화살표류 (→ ← ↔ ↓) | `->` `<-` `<->` `\|` | 다이어그램 포함 |
| 원문자 (① ② ③ / ⓐ ⓑ) | `1)` `2)` `3)` / `a)` `b)` | |
| § (절 기호) | `N절` | "§3-2" -> "3-2절". "§9.2.2" 같은 소수점 절번호는 "9.2.2절" (통째로 옮긴다) |
| … (말줄임) | `...` | |
| × ÷ ² ° ≈ ≠ | `x` `/` `^2` `도` `약 ` `!=` | |
| ✅ ❌ (표 안) | `O` `X` | 문장 안이면 서술로 풀기 ("있음", "성공") |
| ⚠️ 💥 ★ ‡ 〃 | 서술로 풀기 | "주의:", "(!)", 각주는 본문 참조로, 〃는 "위와 같음" |
| 트리 문자 (├ └ │ ─ ┼) | `\|--` `` `-- `` `\|` `-` `+` | `tree --charset=ascii` 스타일 |
| µ | 원어 표기 유지 또는 현행 명칭 | 예: µFeatures -> uFeatures 또는 현행 명칭(TMA) |
| " " ' ' (둥근따옴표) | `"` `'` | |

## 예외 - 코드 인용 보존 규칙

1. **레포 실코드를 그대로 인용한 블록은 소스와 글자까지 일치시킨다.** 소스 주석에 특수문자가 있으면 인용에도 남긴다 - 인용의 충실성이 우선.
2. 판별 방법: 특수문자가 있는 코드 줄을 실제 소스에서 검색한다. 소스에 같은 줄이 있으면 보존, 없으면(문서가 지어낸 예제/주석이면) 치환.
3. 문서 저자가 단 주석/ASCII 다이어그램은 코드 블록 안이라도 치환 대상이다.
4. `ㅋㅋㅋ` 같은 한글 자모는 키보드 문자다 - 치환하지 않는다 (오히려 사람 티).

## 적용 순서

1. 전수 조사부터: 문서 전체의 비ASCII/비한글 문자를 빈도표로 뽑는다 (`scripts/verify.py --scan`).
2. 기계 치환 가능한 것(표의 윗부분)을 일괄 치환하되 코드 펜스는 언어를 보고 처리한다.
3. 서술로 풀어야 하는 것(⚠️, ★, 〃 등)은 문맥을 읽고 하나씩 처리한다.
4. 치환 후 재조사로 잔존 0을 확인한다.
