# 직역 관용구 블랙리스트 (SSOT)

영어 기술 문서의 관용구를 한국어로 직역해서 생기는 어색한 표현들.
실제 문서 교정에서 독자가 거부한 표현만 등재한다 - 추측으로 늘리지 않는다.

**판별 기준 하나로 통한다: "한국 개발자가 실제로 입에 올리는 말인가?"**
- 영어에서 자연스러운 관용구라도 직역하면 아무도 안 쓰는 말이 된다.
- 반대로 한국어에서 자생한 표현이나 실무에 정착한 번역어는 은유여도 살린다.

## 거부 목록

| 표현 | 원 영어 관용구 | 왜 어색한가 | 대안 |
|---|---|---|---|
| ~라는 세금(을 낸다) | tax | 비용 은유의 직역. 한국어에서 코드에 세금 안 낸다 | 반복되는 코드, 비용 |
| 고아 (Task/프로세스 외 문맥) | orphan | orphan process는 OS 용어지만 일반 문맥 직역은 어색 | 아무에게도 묶이지 않은, 방치된 |
| 하중을 받는 (부품) | load-bearing | 건축 은유 직역 | 핵심은 ~다 |
| 움직이는 과녁 | moving target | 사냥 은유 직역 | ~에 따라 바뀐다 |
| 강등/승격/승급 (개념의) | demoted/promoted/upgraded | 인사 용어를 개념에 적용한 직역 (예: "UserDefaults에서의 승급") | "필수"에서 "선택"이 됐다, 강참조로 잡혔다, Keychain으로 바꿨다 |
| 공짜(로 얻는다/주는) | for free | 긍정문에서 어색 (부정형 "공짜가 아니다"는 자연스러움) | 자동으로, 추가 코드 없이 |
| 1급 (API/시민) | first-class | 직역 어색 | 정식 API |
| 물리 법칙 (제약의 비유) | laws of physics | 과장 직역 | OS 제약, 플랫폼 제약 |
| 집행한다 | enforce | 법 집행 직역 | 강제한다, 맡는다 |
| ~의 심장 | heart of | 신체 은유 직역 | ~의 핵심 |
| 계보 | lineage | 족보 직역. 문맥 따라 허용도 있으나 남용 금지 | 출처, ~에서 온 |
| 종착지 | terminus/end of the road | 여정 은유 직역 | 결론, 최종 형태 |
| 직역 (코드 포팅의 비유) | literal translation | 포팅을 번역에 비유하는 것 자체가 번역투 | 그대로 옮긴 것 |
| 저울 영점 (검사) | calibration | 계측 은유 직역 | (설명으로 풀기: 부품이 고장 나면 위의 테스트가 거짓말한다) |
| 박제된다 | frozen/baked in | 직역 | 고정된다 |
| 관문 | gateway | 문 은유. 남용 시 어색 | 공통 통과 지점, 한곳으로 모음 |
| 의미론 | semantics | 학술 직역. value semantics(값 의미론) 같은 정착 용어만 예외 | 동작, 실행 방식 |
| 규율 | discipline | 훈육 뉘앙스 직역 | 규칙 |
| 능력 (기준/별) | capability | 의인화 직역 | 역할 |
| 해부(한다) | anatomy/dissect | 의학 은유 직역 | 분석, 뜯어보기 |
| ~의 뼈대 | skeleton/backbone | "코드 뼈대"는 허용, "해법의 뼈대"류 추상 결합은 어색 | 핵심 구조, 기본 구조, 기반 |
| 성장 경로 | growth path | 직역 | 확장 경로, 확장 포인트 |
| (보일러플레이트를) 접는다 | fold away | 직역 | 반복 제거, 걷어낸다 |
| 무대가 되는 (코드) | stage | 연극 은유 직역 | 대상 (코드) |
| 사는 곳 | lives in | 의인화 직역 | 위치 |
| 시험대 | test of | 직역 | 사례, 검증 지점 |
| 유령(이 된다) | ghost/zombie | 직역 | 핸들을 잃어 취소할 수 없는 상태 (설명으로 풀기) |
| 편승한다 | piggyback | 직역 | 자동으로 이어진다 |
| 에르고노믹 | ergonomic | API ergonomics의 음차 | 문법 축약, 쓰기 편한 |
| 관통 (주제) | cross-cutting | 직역 | 전 모듈 공통, 공통 관심사 |
| ~세계(로 들어가는) | world (async world) | 직역 | async 작업을 시작할 방법 (설명으로 풀기) |
| 문 (진입의 비유) | door/gateway | 직역 | 방법, 진입점 |
| 기권/거부권 (반환값의 비유) | abstain/veto | 표결 은유 직역 | "재시도할 이유가 없다는 뜻일 뿐" (설명으로 풀기) |
| 냄새(가 난다) | (code) smell | code smell은 정착 용어지만 일반 서술 직역은 어색 | ~라서 비추천 (이유로 풀기) |
| 실해악 | real harm | 한자 조어 | 위험한 진짜 이유 |
| 재등판 | (야구 은유 남용) | 같은 패턴이 다시 나옴을 과장 | 같은 패턴이다 |
| 자매편/자매앱/특별판 | companion piece / sister app | 가족 관계 은유 직역 | 문서는 해당 문서 링크로 대체, 앱은 "같은 팀의 다른 앱" |
| 여정/여행 | journey | 직역 | 과정, 순서 |
| 지뢰(를 밟다) | landmine / step on a mine | 군사 은유 직역. 함정의 실체를 설명하면 은유가 필요 없다 | 나중에 터지는 함정, 이 실패를 겪는 지점 (설명으로 풀기) |
| 문답/예상 질문 프레임 | Q&A format | 형식 자체가 기계 생성 신호 (rules.md 4항 참조) | "정리" 절 + 불릿 |

## 유지 목록 (지우면 안 되는 것)

실무에 정착했거나 원어가 그대로 통용되는 것들. 블랙리스트 패턴과 겉모습이 비슷해도 건드리지 않는다.

| 표현 | 이유 |
|---|---|
| 계약 테스트 (contract test) | 테스트 분야 정착 용어 (Pact, xUnit Patterns) |
| 테스트 대역 (test double) | 번역서에서 정착한 표준 번역 |
| 읽기 경로 / 쓰기 경로 (read/write path) | 저장 시스템 분야 실제 용어 |
| 멱등성 (idempotency) | RFC 기반 표준 번역 |
| single-flight | Go singleflight에서 온 실무 용어. 원어 그대로 쓴다 ("단일 비행" 직역 금지) |
| 게이트, guard | 실무 통용 |
| 파이프라인, 인터셉터, 훅 | 실무 통용 |
| 값 의미론 (value semantics) | Swift 공식 문서 정착 번역. "의미론" 일반 남용과 구분 |
| 보일러플레이트 | 실무 통용 음차 |
| 코드 뼈대, 스캐폴딩 | 구체물(코드 골격)을 가리킬 때는 자연스러움 |
| 공짜가 아니다 (부정형) | 한국어에서 자연스러운 관용 표현 |
| 한국어 자생 설명 장치 | 글쓴이가 만든 한국적 비유(예: 시험지와 응시자, 대기표)는 정의가 함께 있으면 유지. 단 독자가 지적하면 즉시 교체 |

## 운영 규칙

- 이 파일이 유일한 원본(SSOT)이다. 교정 중 독자/사용자가 새 표현을 지적하면 여기에 추가한다.
- 항목 추가 시 "왜 어색한가"에 원 영어 관용구를 반드시 병기한다 - 판별 기준의 근거가 된다.
- 하나가 지적되면 같은 표현을 문서 전체에서 전수 검색해 일괄 처리한다 (rules.md 6항).
